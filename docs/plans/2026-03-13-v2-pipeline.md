# VeloAI v2 — Implementation Plan

> Use `subagent-driven-development` (independent tasks) or `executing-plans` (sequential) to implement this.

**Goal:** Transform VeloAI from a live-API script into a self-hosted cycling data platform with PostgreSQL, Grafana dashboards, and fitness-aware planning.
**Architecture:** Polling ingestor + PostgreSQL + Grafana on homelab Docker; VeloAI CLI on Mac mini reads from DB.
**Tech Stack:** Python 3.9+, PostgreSQL 15, Grafana, Docker Compose, Strava API, Komoot API

```yaml
      - postgres_data:/var/lib/postgresql/data
```

- [ ] Implement CTL/ATL/TSB:
```python
def recalculate_fitness(conn):
    """
    For each date with activities, calculate daily TSS.
    Walk forward day by day, applying EMA:
      CTL = CTL_prev × (1 - 1/42) + tss × (1/42)
      ATL = ATL_prev × (1 - 1/7)  + tss × (1/7)
      TSB = CTL - ATL
    Upsert into athlete_stats.
    """
```

## Task 5: Main scheduler


- [ ] Create `main.py`:
```python
import schedule, time, os
from db import get_connection, create_schema
from strava import refresh_access_token, fetch_recent_activities, backfill
from komoot import sync_routes
from fitness import recalculate_fitness

def poll_strava(conn):
    """Fetch activities since last sync, store streams, recalculate fitness."""

def poll_komoot(conn):
    """Sync routes to DB."""

def run():
    conn = get_connection()
    create_schema(conn)
    backfill(conn, ...)         # first run only, skip if already done

    schedule.every(10).minutes.do(poll_strava, conn)
    schedule.every(1).hours.do(poll_komoot, conn)

    while True:
        schedule.run_pending()
        time.sleep(30)
```

- [ ] Create `Dockerfile`:
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python3", "main.py"]
```

- [ ] Run: `docker compose up --build` → confirm ingestor starts, polls Strava, writes to DB
